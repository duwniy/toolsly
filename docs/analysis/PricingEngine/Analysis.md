# Analysis: Pricing Engine

## 1. Цель
Автоматизация расчета стоимости аренды с учетом динамических наценок и объемных скидок.

## 2. Техническое решение
- **Компонент**: `PricingEngineService`.
- **Входные данные**: `Base Daily Price`, `Start Date`, `End Date`.
- **Логика**:
    - **Weekend Markup**: Итерация по дням аренды. Если день Сб или Вс -> `basePrice * 1.2`.
    - **Bulk Discount**: Если `duration >= 8 days` -> `total * 0.9`.
- **Зависимости**: Только сущность `EquipmentModel` (для получения цены) и даты заказа.

## 3. Безопасность
Расчет производится на бэкенде. Клиент получает `total_price` для ознакомления, но финальная цена фиксируется в момент перехода заказа в статус `RESERVED`.

## 4. Data Flow
1. Пользователь выбирает даты и модель.
2. Фронтенд запрашивает предварительный расчет (`GET /api/v1/orders/quote`).
3. Бэкенд возвращает детализированный расчет (Base, Markups, Discounts).
4. При создании заказа цена сохраняется в таблицу `orders`.
