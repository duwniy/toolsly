# 02. Software Requirements Specification (SRS) — Toolsly

## 1. Функциональные модули
### 1.1. Управление пользователями
- Роли: `RENTER`, `STAFF`, `ADMIN`.
- Атрибут `is_verified`: устанавливается STAFF после проверки документов.

### 1.2. Управление инвентарем
- **Model:** Описание (напр. "Дрель Makita"). Базовая цена, рыночная цена.
- **Item:** Конкретный экземпляр. Статусы: `AVAILABLE`, `RENTED`, `MAINTENANCE`, `IN_TRANSIT`, `BROKEN`.
- **Condition:** Фиксация состояния сотрудником: `NEW`, `USED`, `WORN`, `DAMAGED`, `BROKEN`.

### 1.3. Жизненный цикл заказа (State Machine)
1. `CREATED`: Выбор товаров, расчет цены.
2. `RESERVED`: Включение Soft Lock (15 мин).
3. `ISSUED`: Товар выдан (требуется `is_verified` если сумма > 5000).
4. `RETURNED`: Товар принят, проверка `storage_capacity` филиала.
5. `CLOSED`: Финальный расчет завершен.
6. `CANCELLED`: Бронь снята или заказ отменен.

## 2. Расчет стоимости (Pricing Engine)
Формула: `Total = (Базовая цена * Дни) + Наценка за выходные - Скидка`.
- **Weekend Markup:** +20% к стоимости суток, если день — Сб или Вс.
- **Bulk Discount:** -10% от всей суммы, если срок >= 8 дней.

## 3. Алгоритм Soft Lock
- При переходе в `RESERVED` записывается `reserved_until = now + 15 min`.
- По истечении времени статус `Item` возвращается в `AVAILABLE`, если заказ не перешел в `ISSUED`.

## 4. Нефункциональные требования
- **Консистентность:** Межфилиальный возврат должен атомарно менять локацию `Item` и статус `Order`.
- **Безопасность:** JWT аутентификация, шифрование паспортных данных.
